보안 위협방화벽에 막힌 포트로 서비스 시작 시 외부 통신 실패 → 운영 장애. 반대로 port 만 열고 제거 누락 시 불필요 노출 위험.

대상Rocky Linux 8 / 9 / 10 (KISA 가이드 UNIX 항목)
판단 기준양호 FIREWALL_PORTS 환경변수와 default zone 의 --list-ports 결과가 일치하는 경우, 또는 FIREWALL_PORTS 미지정 (확장 조치 대상 아님)
취약 FIREWALL_PORTS 에 지정한 port/proto 가 default zone 에 허용되지 않은 경우, 또는 형식 오류(<1-65535>/{tcp|udp|sctp|dccp} 위반) 인 경우
조치 방법FIREWALL_PORTS 의 각 항목 형식 검증 후 default zone 에 permanent+runtime 으로 add-port. 이미 등록된 항목은 idempotent skip.
조치 시 영향내부에서만 사용해야 할 포트를 외부에 노출시키면 위험. 가능하면 IP 제한(rich rule, U-28 ALLOWED_HOSTS) 과 병행 적용 권장.
